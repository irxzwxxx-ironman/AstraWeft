# ADR-007：Apache-2.0 与平台支持政策

- 状态：Accepted
- 日期：2026-07-15

## 背景

项目目标是可长期维护和开源发布的桌面工具。许可证、贡献政策、依赖合规与平台支持若延后决定，会影响库选择、发布包和社区预期。

## 决策

- 项目源代码使用 Apache License 2.0，保留 NOTICE。
- 第三方依赖和资产必须记录来源并兼容项目分发；发布生成 SBOM 与许可证清单。
- macOS 是主开发平台。
- Windows 从 Phase 0 起运行同步 CI。
- Linux 从架构起保持兼容，并在公开 Beta 前达到完整支持门禁。
- Python 基线为 3.12 或 3.13；qasync 当前声明 `<3.14`，因此在其正式支持前不宣称 Python 3.14 兼容；依赖通过 uv 锁定。
- 发布包在正式版本前完成 macOS 签名/公证和 Windows 签名方案。
- 遥测默认关闭；若引入必须显式 opt-in 并通过单独 ADR。

## 后果

- Apache-2.0 允许商业和开源使用，同时提供明确专利授权。
- 三平台差异不能推迟到打包阶段，Platform Services 和 CI 必须从早期存在。
- Linux 完整 GUI/打包测试可晚于 macOS/Windows，但不能使用平台专属路径污染 Core。
- 项目名称商标权不由 Apache-2.0 授予；正式公开前仍需完成商标与分发渠道复核。

## 守卫

合并前执行依赖审计；新增二进制、字体、图标或模型资产必须带许可证。平台相关代码通过 Ports/Platform Services 隔离，未验证平台能力不得宣称受支持。

## 替代方案

- GPL/AGPL：更强 copyleft，但与当前广泛插件和商业集成目标不符，未采用。
- 仅支持 macOS：降低早期成本但违背跨平台目标，拒绝。
- 不锁依赖：构建不可复现，拒绝。
